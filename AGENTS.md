# Kynfowk -- Project Instructions

Kynfowk is a family coordination and call scheduling app.

Core product idea:
Families sign up, create a family circle, set availability, and the app helps coordinate recurring or one-off calls based on overlap in schedules. The product should feel warm, simple, and family-centered.

Primary MVP goals:
1. User authentication
2. Family circle creation and member invites
3. Availability collection
4. Call scheduling based on overlapping availability
5. Dashboard showing upcoming calls and recent family activity
6. Connections counter:
   - completed calls
   - total minutes
   - unique family members connected this week
   - weekly streak
7. Case studies page with polished storytelling examples

Technical preferences:
- Use TypeScript
- Prefer Next.js App Router
- Prefer clean server/client separation
- Use Supabase for auth and database
- Keep schema additive and easy to understand
- Use reusable components
- Use warm, modern UI copy
- Avoid overengineering

Coding rules:
- Keep the codebase production-minded but MVP-focused
- Favor clear folder structure and small reusable components
- Add concise comments only where helpful
- Do not introduce unnecessary dependencies
- Create a README with setup instructions
- If database changes are needed, create migrations
- If assumptions are required, choose sensible defaults and keep moving

UX tone:
- Human
- warm
- family-centered
- low-friction
- emotionally intelligent

Important product language:
- Family Connections
- Moments Shared
- Time Together
- Reconnection Streak
- Family Circle

When implementing:
- Start with a brief plan
- Then scaffold the full app structure
- Then implement core MVP flows
- Then summarize changed files and next steps
